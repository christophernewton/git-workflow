Default work flow

Branch from master with feature > do work > dev > staging > master

  • Create new branch from master called feature/new-navigation-menu
  • Commit work as you go
  • Once work is ready to show PM merge into dev
  • Once PM has approved merge into staging for client
  • Once Client has approved merge to master for production

Default work flow with PM change

Branch from master with feature > do work > dev > continue work on branch > staging > master

  • Create new branch from master called feature/new-navigation-menu
  • Commit work as you go
  • Once work is ready to show PM merge into dev
  • Once PM has approved merge into staging for client
  • Once Client has approved merge to master for production

Urgent hotfix

Classified as "site down, needs immediate attentionto continue to run"

Branch from master hotfix > fix issue > master

  • Create new branch from master called hotfix/ie-11-bug
  • Solve issue, if it needs testing go to dev, otherwise deploy to master
  • If it needs testing go to dev, otherwise merge back to master
